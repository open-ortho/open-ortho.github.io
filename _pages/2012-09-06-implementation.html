Once a preliminary version of the standard has been approved, we plan to write an open-source implementation of it. This will serve various purposes:

  1. Help divulge the standard by proving that it is functional.
  2. Build a test application that will aid the testing and debugging of the standard. Every implementer and standard developer could use this application as a reference for functionality testing.
  3. Provide the orthodontic community with a free package that will allow for basic patient database management operations.

The implementation will be in a platform independent language, such as JAVA and most likely hosted on an open source repository site such as sourceforge.net.
